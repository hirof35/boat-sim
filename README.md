# Trig Boat Simulation (C# Windows Forms)

三角関数（Sine波）を応用し、複数の重なり合う波と、その波の高さに合わせてリアルタイムに上下するボートを表現した、2D物理・数学シミュレーションアプリケーションです。

元となった Python (Pygame) 版のロジックを忠実に再現しつつ、C# (Windows Forms) 向けにパフォーマンスと可読性を最適化・改良して移植されました。

## 🚀 主な機能と特徴

- **複合サイン波のシミュレーション**
  周波数（速度）と振幅（高さ）が異なる3つの独立した正弦波（$y = A \sin(\theta)$）をリアルタイムに計算・合成しています。
- **アルファブレンディング（半透明合成）**
  3つの波にそれぞれ透過度（Alpha値）を設定し、それらが重ね合わさることで美しく変化する海の深みを表現しています。
- **動的な当たり判定（ボートの追従）**
  画面中央（$X=300$）において、重なり合った3つの波の中で「最も高い位置（Y座標が最小値）」を毎フレーム動的に検出。緑色のボートが常に一番上の波の海面にぴったりと浮かび、どんぶらことリアルに揺れる挙動を再現しています。
- **最適化された描画処理**
  ダブルバッファリング（`DoubleBuffered = true`）の有効化により、画面のちらつき（フリッカー）を完全に抑え、毎秒30フレーム（30 FPS）の滑らかなアニメーションを実現しています。

## 📦 セットアップと実行方法

1. **プロジェクトの作成**
   Visual Studio を起動し、新しいプロジェクトの作成から **「Windows フォーム アプリ (C#)」** を選択します。

2. **コードの配置**
   自動生成される `Form1.cs`（またはメインとなるフォームクラス）のコードを、本リポジトリのソースコードに置き換えます。

3. **実行**
   `F5` キーを押すか、「デバッグの開始」をクリックすると、`600x600` のウィンドウが立ち上がり、シミュレーションが自動的に開始されます。

## 📝 技術的な仕組み（改良のポイント）

### 1. グラフィックメモリの節約
移植元の Python 版では、波ごとに独立した複数の `Surface`（キャンバス）を毎フレーム生成・クリアして重ね合わせる手法をとっていました。
C#版への改良にあたり、`Color.FromArgb()` を用いて描画色自体にアルファ値を直接持たせる設計に変更。これにより、単一のキャンバス（`Graphics` オブジェクト）への直接重ね描きが可能となり、メモリ消費量とCPU負荷を大幅に削減しました。

### 2. UIスレッドの安全性を考慮したタイマー
アニメーションの駆動には `System.Windows.Forms.Timer` を採用。これにより、描画ロジックと状態更新（角度 $\theta$ の加算）が同じUIスレッド上で安全に実行され、マルチスレッド起因の画面のハングアップや不正なクロススレッド例外の発生を防いでいます。
